# 3. GROWTH & UPDATE PLAN (성장 및 유지보수 계획)

## 3.1 수익화 전략 (Minimalist Approach)
- **"재미 + 가치 공유"**를 최우선으로 하며, 인위적인 상단 노출 광고는 배제함.
- **Support Me (선택적)**: 사이트 하단에 "Buy me a coffee"와 같은 후원 링크만 배치하여 중립성 유지.
- **Passive Growth**: 사이트 자체가 너무 예뻐서 개발자들이 '북마크'하고 싶게 만드는 것이 최고의 수익 전략.

## 3.2 데일리 업데이트 로직 (The Daily Brick)
- **규칙**: 하루에 최소 1개의 새로운 서비스 추가 혹은 기존 서비스 가격 정보 갱신.
- **자동화**: 주요 서비스(Supabase, Vercel 등)의 Changelog나 가격 페이지를 스캔하는 간단한 스크립트를 운영하여 업데이트 알림을 받음.
- **큐레이션**: 단순히 양을 늘리는 게 아니라, "내가 쓰고 싶은 것" 위주로 엄격하게 선별.

## 3.3 바이럴 장치 (The Share Factor)
- **Screenshot-Friendly**: 사이트 어디를 캡처해도 인스타그램이나 트위터에 올리고 싶을 만큼 아름다운 UI 유지.
- **Stack Image Export**: 내가 고른 스택을 하나의 예쁜 '대시보드 이미지'로 다운로드하는 기능.
