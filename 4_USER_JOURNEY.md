# 4. USER JOURNEY (사용자 여정: 영감을 찾는 개발자)

## 4.1 사용자 페르소나
- **이름**: 이현우 (28세, 프론트엔드 개발자)
- **성향**: 기능만큼이나 디자인 디테일을 중요하게 생각함. 새로운 툴을 구경하며 영감을 받는 것을 즐김.
- **상황**: "다음 프로젝트는 진짜 세련되게 만들고 싶은데, 어떤 기술 스택과 UI 라이브러리를 써야 '힙'해 보일까?"

## 4.2 사용자 여정 (The Inspired Flow)
1. **첫 방문 (Aesthetic Shock)**: 사이트 접속 후 Raycast 스타일의 딥 다크 UI와 세련된 레이아웃에 즉각적으로 매료됨.
2. **매일 접속 (Daily Routine)**: "오늘은 어떤 새로운 브릭이 올라왔을까?" 하며 아침 커피와 함께 Daily Tech Feed를 확인.
3. **영감 획득 (Curated Browsing)**: Toools.design 스타일의 필터를 이용해 'Skin(UI)' 카테고리에서 Aceternity UI나 Magic UI 같은 고퀄리티 라이브러리 발견.
4. **시뮬레이션 (Virtual Build)**: 내가 고른 서비스들이 이 예쁜 대시보드 안에서 어떻게 어우러지는지 Stack Builder로 시각적으로 확인.
5. **공유 및 저장 (Appreciation)**: 너무 예쁜 UI를 보며 동료들에게 "이 사이트 진짜 예쁘다"며 링크 공유.

## 4.3 핵심 가치 (Value Point)
- "단순한 정보 검색이 아니라, **아름다운 도구들을 감상하는 즐거움**을 제공하는 것."
