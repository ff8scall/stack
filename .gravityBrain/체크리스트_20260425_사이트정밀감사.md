## 🔍 MASC 분석 결과

### 🎯 PM (프로덕트 매니저) 관점
> "카테고리 페이지(`/category/[slug]`)에 **플레이스홀더 텍스트(`Bricks key to be loaded...`)**가 그대로 노출되고 있다. 이것은 검색 엔진에 발견될 경우 사이트의 전문성을 심각하게 훼손한다. 또한 카테고리 페이지 헤더에 `Best ... Bricks`라는 영어 제목이 하드코딩되어 있어 한국어 유저에게는 어색하다."

### 🏗️ 아키텍트 관점
> "BrickGrid 카드 컴포넌트에서 `Last Updated:`가 하드코딩되어 있다. 또한 상세 페이지의 '연관 도구 비교' 섹션에서 `View Analysis`라는 영어 텍스트가 남아있다. 카테고리 페이지의 도구 설명에 `brick.id`가 그대로 노출되는 건 데이터 바인딩 실수이다."

### 🛡️ 보안/품질 관점
> "카테고리 페이지에서 infra 타입 도구의 가격 표시에 unitPrice를 사용해야 하는데, monthlyPrice 폴백만 처리하여 일부 도구의 가격이 `$undefined/mo`로 표시될 수 있다."

## 📌 발견된 문제 (심각도 순)

|:---|:---|:---|
| `/category/[slug]/page.tsx` | `{brick.id} (Bricks key to be loaded...)` 플레이스홀더 렌더링 | L89 |

### 🟠 P1: 하드코딩 텍스트 잔류
| 위치 | 문제 | 라인 |
|:---|:---|:---|
| `/category/[slug]/page.tsx` | `Best ... Bricks` 영어 제목 하드코딩 | L62 |
| `/category/[slug]/page.tsx` | `2026년 현재 가장 뛰어난...` 한국어 하드코딩 (영어 미지원) | L65 |
| `/category/[slug]/page.tsx` | `GO →` 영어 하드코딩 | L95 |
| `/category/[slug]/page.tsx` | 가격 표시에서 infra/request 타입 미처리 | L93 |
| `BrickGrid.tsx` | `Last Updated:` 하드코딩 | L161 |
| `/brick/[id]/page.tsx` | `View Analysis` 영어 하드코딩 | L329 |

### 🟡 P2: 카테고리 페이지 완성도 부족
- [ ] 카테고리 페이지에서 각 도구의 번역된 이름과 설명을 사용하지 않고 `brick.name`(원어)과 플레이스홀더를 사용
- [ ] 카테고리 페이지에 Navbar 컴포넌트 누락 (다른 페이지들은 모두 포함)
- [ ] 카테고리 페이지에 Footer 컴포넌트 누락

### 🟢 P3: 디자인 개선 여지
- [ ] 카테고리 페이지 카드에서 도구 상세 페이지(`/brick/[id]`)로의 직접 링크 부재 (현재 메인 페이지로만 이동)
- [ ] 카테고리 페이지 카드에 새로 추가한 `description` 데이터 활용 가능

## 📋 실행 계획

### Phase A: 긴급 수선 (~10분)
1. 카테고리 페이지 플레이스홀더 텍스트 제거 → 번역된 도구 설명(`description`) 사용
2. 카테고리 페이지 가격 표시에 모든 pricing type 처리 추가

### Phase B: i18n 완결 (~15분)
3. 카테고리 페이지 헤더 다국어화 (`Best ... Bricks` → 번역 키)
4. 카테고리 페이지 설명 다국어 분기 추가
5. `BrickGrid.tsx`의 `Last Updated:` 번역 키 전환
6. 상세 페이지 `View Analysis` 및 `연관 도구` 섹션 다국어 전환
7. 카테고리 페이지 `GO →` 번역 키 전환

### Phase C: 카테고리 페이지 완성도 향상 (~10분)
8. Navbar 및 Footer 추가
9. 카드 링크를 상세 페이지(`/brick/[id]`)로 변경
10. 카드에 번역된 도구명 및 description 사용
