# LegoStack STRATEGY_MAP (전략 지도)

## 1. 서비스 위계 (Hierarchy) - v2
- **Layer 0: First Impression (Hero)** - 3초 안에 "AI 비용 계산기"임을 인지.
- **Layer 1: Discovery (Main Grid)** - 다양한 AI 브릭을 발견하고 DX 점수를 확인.
- **Layer 2: Selection (Add to Stack)** - 필요한 브릭을 스택에 추가 (조립 행위).
- **Layer 3: Projection (Calculator Bar)** - MAU 조절을 통한 미래 비용 예측.
- **Layer 4: Comparison (A vs B)** - 도구 간 직접 비교로 의사결정 가속.
- **Layer 5: Viral (Sharing)** - 시뮬레이션된 견적서를 URL로 공유.

## 2. 사용자 인지 여정 (Cognitive Journey) - v2
- **Impact** -> 히어로의 실시간 비용 카운터에 눈이 멈춤 (3초 규칙)
- **Curiosity** -> "와 예쁘다, 이게 뭐지?"
- **Interaction** -> 프리셋 또는 검색으로 빠른 체험
- **Discovery** -> "슬라이더를 밀면 비용이 이렇게 변한다고?"
- **Value** -> "내 서비스 규모에는 이 조합이 딱이네!"
- **Advocacy** -> 스택 공유 → 신규 유저 유입

## 3. 핵심 전략 목표 (v2)
- **Acquisition**: 히어로 + SEO 비교 페이지 + GEO 대응으로 오가닉 유입 극대화.
- **Activation**: 프리셋 원클릭 체험 및 검색 기능으로 첫 시뮬레이션까지의 시간 단축.
- **Retention**: 정확한 가격 데이터 및 '새로운 도구 추가' 알림으로 재방문 유도.
- **Referral**: 공유 스택 프로필 + OG 이미지 + 갤러리로 바이럴 루프 완성.
- **Authority**: AI 인프라 비용 계산의 '표준 도구'로 자리매김 (GEO 최적화).

## 4. 유입 채널 우선순위
1. SEO (롱테일 비교 키워드) - 최우선
2. 커뮤니티 (Reddit, HN, Dev.to) - 런칭 부스트
3. SNS 바이럴 (공유 스택) - 지속적
4. AI 디렉토리 등록 (TAAFT, Futurepedia) - 백링크 확보
5. GEO (AI 검색엔진 인용) - 장기 전략
