# 체크리스트: Phase 3 전략 보완 및 기술적 구현

## 📋 작업 개요
- **목표**: 1인 개발자의 도구 탐색을 넘어, 공유와 바이럴이 일어나는 'AI FinOps 시뮬레이터'로서의 기능을 완성한다.
- **중점 사항**: Short Slug 기반 공유, OG 이미지 자동 생성, 현실 세계 수치 변환 UI.

## 🛠️ 세부 작업 항목

### 1. Hybrid Short Slug 시스템 구축
- [ ] **DB 스키마 설계**: Supabase 내 `shared_stacks` 테이블 생성 (id, slug, stack_data, clicks, created_at).
- [ ] **API Route 개발**: `app/api/share/route.ts` - 상태 데이터 압축 저장 및 슬러그 발급 로직.
- [ ] **Dynamic Routing**: `app/[locale]/s/[slug]/page.tsx` - 슬러그로부터 상태 복원 및 렌더링.
- [ ] **LZ-String 도입**: 데이터 압축을 통한 저장 용량 및 전송 효율 최적화.

### 2. 시각적 바이럴 엔진 (Viral Image Card)
- [ ] **html-to-image 연동**: 시뮬레이션 결과 요약 섹션을 이미지로 변환하는 유틸리티 개발.
- [ ] **동적 OG 태그**: 공유 URL 접속 시 해당 스택의 주요 비용 정보가 포함된 메타 태그 생성.
- [ ] **공유 버튼 고도화**: 트위터, 링크드인, 카카오톡 특화 공유 링크 생성 및 이미지 첨부 가이드.

### 3. Contextual UI (해석 도우미)
- [ ] **매핑 데이터 구축**: MAU 및 Usage 수치별 현실 예시(유동인구, 책 분량 등) 데이터셋 작성.
- [ ] **슬라이더 컴포넌트 수정**: `CalculatorBar` 내 슬라이더 상단에 실시간으로 해석 레이블을 보여주는 애니메이션 UI 추가.
- [ ] **다국어 지원**: 각 언어별(ko/en)로 문화권에 맞는 적절한 비유(예: 강남역 vs 타임스퀘어) 적용.

### 4. 데이터 자산화 및 분석
- [ ] **트래킹 로직**: `api/redirect` 핸들러 내에서 유입 슬러그 정보 기록 기능 추가.
- [ ] **인기 스택 집계**: DB 데이터를 기반으로 메인 페이지 하단에 '인기 있는 스택 조합' 섹션 구현 검토.
