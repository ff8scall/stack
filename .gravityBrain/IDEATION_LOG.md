# LegoStack IDEATION_LOG (기획 로그)

## 2026-04-25: Grand Strategy 전면 진단 및 유입 전략 수립
- **발단**: 사용자가 현행 사이트의 전반적인 보완/수정 사항을 심도있게 검토 요청 (유료 제외, 유입 최우선).
- **분석 방법**: 6단계 Grand Strategy (/research → /brainstorm → /monetize → /user-journey → /pre-verify → /mvp-scope) 순차 실행.
- **치명적 발견**:
  1. `presets.ts`의 3개 프리셋 모두 존재하지 않는 brick ID를 참조 → 핵심 기능 고장 상태.
  2. 모바일 Navbar 카테고리 탭이 세로로 쌓여 레이아웃 완전 깨짐.
  3. 히어로 섹션 부재로 첫 방문자 가치 전달 실패.
  4. 20+ 하드코딩 영어 텍스트가 i18n 미처리 상태.
- **핵심 결정**:
  - Phase 4를 A~D 4단계로 세분화하여 우선순위별 실행.
  - P0(긴급수선) → P1(유입기반) → P2(유입폭발) → P3(권위확립) 순서.
  - SEO 비교 페이지 자동 생성을 최우선 유입 전략으로 채택.
  - GEO(AI 검색엔진 최적화)를 장기 전략으로 설정.
- **기대효과**: P0 수선만으로도 기능 복구 + 모바일 대응이 해결되며, P1의 히어로 섹션 추가 시 첫 방문자 이탈률 50% 이상 감소 예상.

## 2026-04-22: 직관성 강화 및 조립 비주얼 전략 수립
- **발단**: 상세보기 기능 추가 후, 사용자가 메인 그리드와 계산기 바 사이의 연결 고리를 "어렵다"고 느낌.
- **분석**: 카드 선택 행위가 '조립'이라는 컨셉으로 전이되지 않고 단순 '체크'로 인식됨.
- **결정**: 
  - `CalculatorBar`를 단순 결과창이 아닌 'Stack Dock'으로 재정의.
  - 선택된 브릭들이 하단 바에 시각적으로 쌓이는 연출(Framer Motion) 강화.
  - '상세보기'는 도구의 스펙 확인용, '메인 그리드'는 조립용으로 역할을 분담.
- **기대효과**: "조립 -> 예측"으로 이어지는 서비스의 본질적 가치를 사용자가 별도의 설명 없이도 깨닫게 함.
