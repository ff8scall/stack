# LegoStack CONCEPT_LOGIC (개념 로직 정의)

## 1. 핵심 개념: Visual Infrastructure Assembly
LegoStack은 단순히 AI 도구를 나열하는 디렉토리가 아닙니다. 사용자가 자신의 서비스 규모(MAU, Usage)에 맞춰 인프라를 '조립'하고, 그에 따른 '비용 편익'을 실시간으로 시뮬레이션하는 **Interactive FinOps Playground**입니다.

## 2. 브릭(Brick) 시스템 원리
- **단일성**: 각 도구는 독립적인 '브릭'으로 존재하며, 고유의 가격 데이터(PricingData)를 가집니다.
- **조립성**: 사용자가 브릭을 선택하면 'Stack'이라는 가상 환경에 추가됩니다.
- **반응성**: MAU 또는 사용량(Usage) 슬라이더가 변하면, 스택에 포함된 모든 브릭의 비용이 재계산되어 총합(Total Cost)에 반영됩니다.

## 3. 직관성 강화 로직 (Strategic Update 2026-04-22)
- **Visual Stacking**: 하단 CalculatorBar는 선택된 브릭들의 시각적 적재소(Dock) 역할을 수행합니다.
- **Cost Sensitivity**: 가격 변화를 단순히 숫자가 아닌 '비중 변화'로 보여주어, 특정 인프라의 가격 민감도를 직관적으로 인지하게 합니다.
- **Contextual Anchoring**: "동네 카페 방문객 수"와 같은 비유를 통해 추상적인 MAU 수치를 현실적인 규모감으로 치환합니다.

## 4. 유입 극대화 전략 (Grand Strategy Update 2026-04-25)
- **3초 규칙**: 히어로 섹션의 실시간 카운터 애니메이션으로 첫 방문자에게 3초 안에 서비스 가치 전달.
- **비교 페이지 (A vs B)**: SEO 롱테일 키워드 지배를 위해 모든 동일 카테고리 브릭 쌍의 비교 페이지를 정적 생성.
- **GEO 대응**: JSON-LD 구조화 데이터 + Q&A 형식 콘텐츠로 AI 검색엔진이 LegoStack을 '표준 도구'로 인용하도록 유도.
- **바이럴 루프**: 공유 스택 → 프로필 페이지 → 신규 유저 → 자체 스택 생성 → 재공유의 선순환 구조.
